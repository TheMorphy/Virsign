# Virsign

# Тестовое задание — симуляция вилочного погрузчика

В рамках тестового задания реализована мини-игра, в которой необходимо перевезти груз со стартовой площадки на площадку разгрузки с помощью вилочного погрузчика.

Общее время разработки: **примерно 12–14 часов**

Проект выполнен на **Unity 6000.2.13f1** с использованием **новой Input System, Zenject, R3, DoTween, Cinemachine и др. сторонних плагинов**, а также архитектурных практик для разделения игровой логики.

# Реализованный функционал

Реализованы все механики, указанные в техническом задании.

### Запуск сцены

* сцена начинается с **просветления экрана из черного цвета**

### Управление погрузчиком

* управление с клавиатуры
* движение реализовано на **Rigidbody физике**
* управление WASD

### Двигатель

* двигатель запускается клавишей **T**
* пока двигатель не запущен:

  * движение погрузчика отключено
  * управление вилами недоступно

### Вилы погрузчика

* подъем / опускание при удержании **Q / E**
* реализовано с использованием **ConfigurableJoint**

### Груз

* груз представлен на поддоне
* в начале сцены груз:

  * **появляется над стартовой площадкой**
  * плавно опускается вниз
  * вращается вокруг вертикальной оси
  * анимация длится ~5 секунд

### Площадка разгрузки

* при доставке груза на площадку разгрузки:

  * груз **взлетает вверх**
  * вращается по всем осям
  * ускоряется
  * анимация длится ~5 секунд

### Камера заднего вида

* реализована **камера заднего вида**
* изображение выводится на монитор в кабине погрузчика

### Система топлива

* во время движения расходуется топливо
* когда уровень топлива падает ниже **50%**:

  * скорость погрузчика уменьшается **в 2 раза**

### Индикатор топлива

* уровень топлива отображается в кабине
* реализован визуальный индикатор уровня

# Используемые технологии

## Новая Unity Input System

Используется новая система ввода Unity.

Позволяет:

* гибко настраивать управление
* легко расширять поддержку устройств
* централизовать обработку ввода.

## Zenject

Используется для управления зависимостями.

Преимущества:

* слабая связность компонентов
* удобное управление жизненным циклом сервисов
* более масштабируемая архитектура.

## R3 (Вместо UniRX)

Используется для реактивной обработки состояния.

Например:

* изменение состояния двигателя
* обновление UI
* реакции на уровень топлива.

## ScriptableObject конфигурации

Игровые параметры вынесены в **ScriptableObject**.

Например:

* параметры движения погрузчика
* параметры топлива
* параметры вил

Это позволяет легко изменять баланс без изменения кода.

## DOTween

Используется для анимаций:

* появление груза
* выгрузка груза
* UI эффекты
